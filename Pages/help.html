What is PlayBlocker?

PlayBlocker is a stage-blocking tool for theatre directors. You upload a floor plan of your set and an HTML-formatted script, place your actors on the diagram, then annotate the script with movement markers as you work through the blocking session.

Each movement records where an actor was standing at the start of a line, any intermediate positions they pass through, and where they end up. All positions are saved to the server so the session can be resumed later exactly as you left it.

The ? badges

When you open a production in the blocking workspace you will see small ? badges in the corners of the stage image panel and the script panel. These are context-sensitive help triggers.

  • Hover over a ? badge to display a short hint about that area in the hint bar at the bottom of the workspace.
  • The hint bar also updates automatically as you work — for example, it reminds you to drag an actor once a movement has been started, or confirms what a right-click will do at the current cursor position.
The ? badges are always there if you need a quick reminder. They don't interrupt your work — just hover and the hint appears in the bar below.

Productions page

The Productions page is where you create and configure each show before opening it in the blocking workspace.

Creating a production

Click + New Production in the left panel, give the production a name, and click Save.

Stage image

Click Upload Image to upload a floor plan, set diagram, or any image that represents the stage. PNG and JPEG work well. The image is stored on the server and reloaded automatically each session.

Script

Speakers

Once a script is loaded, PlayBlocker extracts the cast from it and lists them in the Speakers table. For each speaker you can set:

  • First / Last name — displayed in alerts and tooltips.
  • Initials — the two-letter code shown on the stage diagram and inside movement markers in the script.
  • Color — the fill color of the speaker's icon. Click the color box to choose from a palette.

Click Save Speakers to persist your changes, then click Open in PlayBlocker to begin blocking.

Stage image

The top portion of the blocking workspace shows the stage image. Actor icons are placed here and move in real time as you navigate through the script.

  • Drag an actor icon from the speaker panel onto the stage image to place them on stage for the first time.
  • An actor must be on stage before a movement can be recorded for them.
  • Actor positions are saved automatically whenever a speaker is dropped.
  • Positions are stored as proportions of the image size, so they remain accurate if you resize the browser window.
Right-click an on-stage actor icon to enter Edit mode for their most recent movement before the cursor.

Speaker panel

The speaker panel sits to the right of the stage image. Each cast member has a colored circular icon showing their initials.

  • Drag an icon onto the stage image to place the actor on stage.
  • Hold the mouse button down on an on-stage icon to preview the path of their most recent movement before the current cursor position. Release to dismiss.
  • Icons that have not yet been placed on stage cannot be dragged as part of a movement — place them first.

Script area

The script is shown in a scrollable panel below the stage image. Movement markers appear inline in the text.

Cursor position

A blinking blue cursor shows your current position in the script. The stage image always reflects the actor positions recorded at the nearest movement before the cursor.

  • Left-click anywhere in the script to move the cursor there. Actor icons reposition immediately.
  • If clicking would jump to a different movement zone — changing the actor layout — a confirmation prompt appears before the change is applied.

Movement markers

Completed movements appear as inline spans like [AB], where the letters are the actor's initials. Clicking elsewhere in the script navigates relative to these markers.

Keyboard navigation

  • Tab — advance the cursor to the next Speech paragraph.
  • Shift+Tab — move the cursor back to the previous Speech paragraph.
  • ← — jump the cursor to immediately after the nearest movement marker to the left.
  • → — jump the cursor to immediately before the nearest movement marker to the right. If the cursor is already adjacent to that marker, move past it and display the actor's path for two seconds. Press any key to dismiss the path early and continue.
  • Ctrl+← — move to the beginning of the current sentence. If already at a sentence start, move to the start of the previous sentence. If at the beginning of a Speech paragraph, move to the last sentence of the preceding Speech paragraph.
  • Ctrl+→ — move to the beginning of the next sentence. If in the last sentence of a Speech paragraph, move to the beginning of the next Speech paragraph.
  • ↑ ↓ Page Up Page Down — scroll the script; the cursor repositions automatically after scrolling stops.
  • Escape — cancel a movement that has been started but not yet completed.
  • Space — while dragging an actor icon during movement creation, freeze a waypoint marker at the current position.

Page slider

The slider at the bottom of the workspace lets you jump quickly to any page of the script. Drag the thumb to scrub through the script; the cursor repositions automatically when you release.

The page number display to the left of the slider updates in real time as you scroll.

Recording a movement

A movement records the path an actor takes during a specific line of dialogue or stage direction.

  1. Left-click in the script to position the cursor at the line where the movement begins. The stage image will show actor positions at that point.
  2. Right-click on the Speech paragraph where the movement starts. A [?] marker appears in the text.
  3. If the cursor has moved to a different section of the script and the actor layout would change, confirm the prompt to proceed.
  4. Drag the actor who is moving. The [?] updates to their initials, and a ghost icon appears at their starting position.
  5. While dragging, press Space to drop a waypoint marker at any intermediate position along the path.
  6. Drop the actor icon on their destination on the stage image. The movement is saved automatically and the marker becomes [AB].
Only actors who are already on stage can be assigned a movement. Drag any unplaced actor onto the stage image first.

Editing a movement

You can reposition the start, end, and waypoints of any saved movement.

  1. Left-click in the script to place the cursor at or after the movement you want to edit.
  2. Right-click the actor's icon on the stage image and choose Edit [Name]'s movement.
  3. A ghost icon appears at the movement's starting position. Drag it to a new starting position.
  4. Drag the actor's icon to a new ending position.
  5. Drag any of the waypoint markers to reposition intermediate points.
  6. Right-click any of these elements and choose Save to persist the changes, or Cancel to discard them.
Moving the end position of a movement automatically updates the starting position of the same actor's next movement, keeping the path continuous.

Previewing a movement path

To quickly remind yourself of an actor's most recent movement without entering edit mode, hold the left mouse button down on their icon. The ghost icon and waypoints for that movement appear on the stage. Release the button to dismiss them.

The movement shown is always the one immediately before the current cursor position in the script.

Saving

Click the Save button in the top bar of the blocking workspace to save the current state of the annotated script to the server. This preserves the cursor position, all movement markers, and the current actor layout so the session can be resumed exactly as you left it.

Individual speaker positions and new movements are saved to the server automatically as you work, so you will not lose blocking data if you forget to click Save — but the cursor position and the overall script state are only saved when you click the button.

The browser will warn you if you try to close or navigate away from the page with unsaved changes.
